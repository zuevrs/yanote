## FlatDir bundle for yanote recorder (design)

**Цель:** дать самый быстрый способ проверить, что `yanote-recorder-spring-mvc` действительно пишет `events.jsonl` в “закрытом” контуре, подключаясь к сервису как набор локальных JAR’ов (Gradle `flatDir`), без публикации в репозитории и без спец‑заголовков.

### Контекст

У пользователя есть Spring Boot 3.x сервис (Gradle Groovy), который запускается отдельным процессом/контейнером, и E2E тесты/проверки, которые дергают REST эндпоинты. Нужно максимально быстро убедиться, что при любых HTTP вызовах рекордер создаёт и пополняет `events.jsonl`.

### Принятое решение (recommended)

- **Вариант C (flatDir + fileTree)**:
  - собрать `yanote-recorder-spring-mvc` и `yanote-core` в этом репо
  - сформировать “bundle” папку `dist/flatdir-recorder/`
  - складывать JAR’ы (как артефакт сборки, не коммитить) в `dist/flatdir-recorder/libs/`
  - в целевом сервисе подключать все JAR’ы из `libs/yanote/*.jar` через `flatDir` + `fileTree`
- **Включение записи только при проверке**:
  - запускать сервис с `yanote.recorder.enabled=true`
  - задавать `yanote.recorder.events-path` на путь в writable volume, чтобы легко забрать `events.jsonl` наружу
- **Smoke-проверка**:
  - поднять сервис с рекордером
  - сделать любой HTTP запрос
  - проверить, что `events.jsonl` появился и не пустой

### Почему так

- Это самый быстрый путь для закрытой сети: не нужен внутренний Maven/Nexus, не нужен `mavenLocal()`, не нужно синхронизировать публикацию артефактов.
- Для цели “видим запись REST вызовов” достаточно автоконфигурации рекордера и файла `events.jsonl`.

### Риски / ограничения

- `flatDir` не управляет транзитивными зависимостями: bundle должен включать все “не-spring” зависимости, которых нет в сервисе.
- Возможны конфликты версий (особенно если случайно принести в `libs` Spring Boot/Spring Web). В bundle лучше не включать Spring/Boot артефакты, полагаясь на те, что уже есть в сервисе.

### Success criteria

- Разработчик может выполнить команду сборки bundle в этом репо и получить папку `dist/flatdir-recorder/libs/` с JAR’ами.
- В целевом сервисе подключение этих JAR’ов + запуск с `yanote.recorder.enabled=true` приводит к появлению и росту `events.jsonl` при HTTP вызовах.
